# Soul Design — How to Write an Effective SOUL.md

> Your agent's SOUL.md is its constitution. Everything else flows from it.

## Principles

1. **Identity before capability** — Define who the agent is before what tools it has
2. **Boundaries are features** — Clear limits make the agent more trustworthy, not less capable
3. **Rules must be enforced** — An unenforced rule is worse than no rule (it creates false confidence)
4. **Evolve through incidents** — Every mistake should produce a new rule

## Structure

A good SOUL.md has these sections (in order):
1. Identity & Role
2. Communication Style
3. Response Rules
4. Security Boundaries (highest priority)
5. Prompt Injection Defense
6. Identity Protection
7. Error Correction Mechanism

## Common Mistakes

- Writing rules "for show" that aren't enforced
- Being too generic ("be helpful") instead of specific ("conclusion first, under 800 chars in group chat")
- Forgetting to add identity protection (agent reveals it's AI when asked)
- No error correction loop (same mistakes repeat)
